class aes128_agent extends uvm_agent
A UVM agent containing an appropriate sequencer, driver, and monitor in order to stimulate the DUV and observe the resulting outputs.
class aes128_comparator extends uvm_agent
The comparater will receive the actual responses from the monitor and the expected responses from the predictor.
class aes128_config extends uvm_object
A UVM configuration object to be used by the test environment of the AES-128 example.
class aes128_driver extends uvm_driver #(txn_request)
The driver receives the sequences from the sequencer and translates it into the “pin wiggles” required by the DUV, which is connected via the virtual interface to the driver.
A UVM environment for the AES-128 example.
class aes128_monitor extends uvm_monitor
The monitor is an absolutely passive UVM component, which solely observes the communication to and from the DUV.
class aes128_predictor extends uvm_agent
Usually, the predictor may use the obtained input data to the DUV (received from the monitor) in order to feed it into a “golden model”.
A test, which takes reads the stimuli from a file, applies it to the DUV, observes the resulting output and compares the actual responses against the expected ones, which are also read from a file.
class mbgh_report_server_c extends uvm_report_server
This class provides a custom UVM report server in order to provide a nicer log output in the simulator when using the UVM reporting macros.
class seq_from_file extends uvm_sequence #(txn_request)
This sequence reads the stimuli from an earlier prepared file and passes the transactions on to the driver, which then drives the actual DUV.
A transaction containing only the data being transferred between the DUV and its environment.
A transaction containing both the data and the controlling signals of the AES-128 DUV.
